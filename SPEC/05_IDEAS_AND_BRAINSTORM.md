### 核心原則 (Core Values)
1. **透明性**：員工清楚知道系統在監測什麼。
2. **非侵入**：追蹤應用程式使用時間與視窗標題，不截圖不記錄鍵盤。
3. **輕量化**：最小化系統資源佔用。
4. **隱私保護**：不截圖、不追蹤網頁內容、不記錄按鍵。

### 🔒 隱私保護細項 (Privacy Statement)
- **記錄項**：App 名稱、視窗標題、使用時長、電腦名稱、時間。
- **絕對不記錄**：螢幕截圖、鍵盤輸入、瀏覽網址 (URL)、文件具體內容、私人帳號密碼。

## 🛠️ 技術選型與決策史 (Technical Context)

### 1. 軟體堆疊檔案塊 (Tech Stack)
| 技術 | 用途 |
| :--- | :--- |
| **Electron** | 跨平台桌面框架 (v28.x) |
| **Node.js** | 執行環境 (v20.x LTS) |
| **SQLite** | `better-sqlite3` / `sql.js` 雙引擎本地存儲 |
| **Google Apps Script** | API 伺服器與 Sheet 邏輯 |
| **Firebase RTDB** | WebSocket 即時訊息推送 (asia-southeast1) |

### 2. 通訊方案對比 (Decision History)
- **方案 A (輪詢 Polling)**：每 3 分鐘抓取一次。缺點是 API 開銷高 (10台電腦 = 4800次/天)，延遲大。
- **方案 B (Firebase 即時推送 ✅)**：採用 WebSocket 監聽。延遲 < 1s，API 開銷極低。這是目前系統選擇的最終方案。
