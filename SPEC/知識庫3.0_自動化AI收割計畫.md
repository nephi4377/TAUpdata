> [!IMPORTANT]

## 1. 核心願景與產出指標
本系統旨在解決室內裝修業「知識斷層」問題。透過自動化採集，目標在 30 天內建立超過 500 筆高品質、具法規依據的知識條目。
- **產出格式**：標準化 Markdown 知識卡片。
- **目標數量**：首批生成 300+ 條目，涵蓋泥作、水電、木作、油漆、廚衛。
- **品質標準**：具備具體數值（如間距、厚度、磅數）、法規依據（如建築技術規則章節）及總監級專業對策。

## 2. 絕對獨立性協議 (Isolation Protocol) - 極限隔離架構

為徹底杜絕「交叉感染」（即爬蟲庫與小助手核心庫衝突），系統採用 **完全物理隔離** 設計：

### A. 獨立運算空間 (Sandbox)
- **路徑**：`.../CodeBackups/CODING/KnowledgeFactory/`
- **獨立環境**：該目錄擁有專屬的 `package.json` 與 `node_modules`。
- **禁用對象**：嚴禁引用 `client/src/` 下的任何 Electron 模組或 Node 內建 bridge 文件，防止 Webpack/Vite 聯動編譯錯誤。

### B. 單向通訊協議 (One-Way Bridge)
- 兩系統間**無動態通訊 (No RPC/Socket)**。
- 數據僅透過 `KnowledgeFactory/output/` -> `client/KNOWLEDGE/pending_review/` 的 **JSON 文件物理搬移** 進行對接。

## 💡 白話文版：收割機是如何運作的？ (五大詳細步驟)

您可以把這個系統想像成一個 **「動作極快、永遠不會累的讀書小秘書」**：

1.  **第一步：進門 (瞄準目標)**
    *   小秘書會先來到您指定的資料庫，比如「100室內設計」的百科首頁。它會先看一眼今天有多少新文章，確認目標座標。
2.  **第二步：翻頁 (搬出所有的書)**
    *   一般人看網頁要一頁一頁點，很累。小秘書會直接「瞬移」，從第 1 頁、第 2 頁一路翻到第 100 頁，把所有感興趣的文章連結全部記錄下來。
3.  **第三步：摘錄 (只看您要的內容)**
    *   小秘書會點進每一篇文章裡。它很聰明，它會**無視**旁邊的廣告、推薦連結或無連的留言。它眼裡只有這篇文章的**「標題」**與**「內文規範」**。
4.  **第四步：去骨 (清洗資料)**
    *   網頁上有很多雜亂的代碼。小秘書會把這些「骨頭」剔掉，只留下最純淨的**文字內容**，就像把雜亂的草稿整理成乾淨的逐字稿。
5.  **第五步：存進臨時倉庫 (待命)**
    *   最後，它會把今天抓到的幾百篇文章，整整齊齊地擺在一個叫 `raw_data` 的臨時倉庫裡。這時候資料還沒進小助手，而是安靜地躺在隔離區，等待下一位 **AI 處理師** 來把它們濃縮成精華。

**一句話總結**：爬蟲就是負責 **「出國掃貨、整箱搬回來」** 的採購員；而後面的 AI 則是負責 **「精挑細選、貼標籤入庫」** 的首席店長。

## 3. 爬蟲階段深度規格 (Deep Dive: Harvester)

### A. 目標站點選擇與權限
1. **100室內設計百科**：重點抓取「裝修工法」與「建材規格」。
2. **設計家 Searchome**：重點抓取「QA 問答」與「維護手冊」。
3. **全國法規資料庫**：針對《建築物室內裝修管理辦法》進行條文對應。

### B. 技術實作細節
- **併發控制**：使用 `p-limit` 設定 `CONCURRENCY = 3`。每分鐘請求不超過 20 次，模擬真人行為，避免觸發 WAF 限流。
- **解析邏輯 (Extraction Schema)**：
    - `title`: 抓取 `<h1>` 或 `.article-title`。
    - `raw_content`: 使用 `turndown` 將 `.entry-content` 轉換為 Markdown，過濾掉所有 `<img>` 標籤以防盜鏈報錯。
- **網頁清洗 (Sanitization)**：移除廣告、推薦連結與側欄導喚連結。

## 5. 數據流與存檔方式 (Data Flow & Storage)

系統採用「無塵過濾」式的存檔流程：

1.  **外部抓取**：爬蟲在隔離區運作，將網頁內容存為 `raw_data/*.json`。
2.  **AI 萃取**：處理器將原始資料精煉後，輸出純乾淨的 `structured_cards/*.json`。
3.  **單向匯入**：手動或腳本將 JSON 拷貝至小助手 `KNOWLEDGE/pending_review/`。
4.  **總監審核存檔**：
    -   **位置**：統一存入專屬檔案 `client/KNOWLEDGE/知識庫_AI自動生成.md`。
    -   **方式**：採用 Markdown **追加 (Append)** 模式。
    -   **優點**：即使 AI 存檔出錯，也僅影響該單一 MD 檔，不會導致小助手主程式崩潰。

## 5. 如何重啟與執行 (How to Implement)

若未來需要重啟此自動化管線，請依照以下步驟操作：

### A. 環境準備
1.  **進入獨立工廠目錄**：`cd .../CodeBackups/CODING/KnowledgeFactory/`
2.  **安裝依賴** (僅在此目錄執行)：
    ```powershell
    npm init -y
    npm install axios cheerio turndown openai p-limit
    ```

### B. 執行收割與處理 (Turbo Mode)

#### 1. 收割階段 (Harvester Logic)
執行命令：`node scrapers/harvester.js --turbo`

**具體執行步驟：**
1.  **種子 URL 設定**：爬蟲讀取預設的專業百科首頁（如：100室內設計百科）。
2.  **分頁抓取 (Pagination)**：
    *   循環遞增 `page` 參數（例如 `?page=1...50`）。
    *   使用 `p-limit` 控制併發請求量，避免被目標伺服器封鎖 (IP Block)。
3.  **選擇器解析 (DOM Selection)**：
    *   利用 `cheerio` 定位條目：例如：`.article-list li a` 提取所有文章連結。
    *   **深度爬取**：進入子分頁抓取 `h1` (標題) 與 `.article-content` (全文)。
4.  **網頁清洗 (Sanitization)**：
    *   移除廣告、腳註、側欄導喚連結。
    *   將 HTML 轉換為乾淨的 Markdown 格式（使用 `turndown`）。
5.  **原始數據持久化**：
    *   產出格式：`{ source: string, url: string, raw_text: string, timestamp: number }`。
    *   存檔位置：`raw_data/harvest_yyyymmdd.json`。

#### 2. AI 處理階段 (Processor Logic)
執行命令：`node processor/processor.js`

### C. 匯入與審核
1.  **手動/自動同步**：將 `output/` 下的 JSON 檔案複製到小助手的 `client/KNOWLEDGE/pending_review/`。
2.  **介面啟動** (需還原 `monitor.js` 相關代碼)：管理員 Level 5 帳號點擊「🚨 知識待核」即可開始批閱。

## 6. 失敗經驗與防錯 (Lessons Learned)
- **嚴禁交叉感染**：爬蟲用的 `node_modules` 絕對不可進入小助手的 `client/` 目錄，否則會導致編譯錯誤。
- **字串轉義陷阱**：在 `monitor.js` 中渲染 JSON 時，需嚴格處理引號逃逸 (`.replace(/'/g, "\\'")`)，以防語法潰散。
